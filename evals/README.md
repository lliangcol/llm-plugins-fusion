# Workflow Evaluations

These public-safe fixtures are separated into three evidence layers:

- `route/cases.json` and `baselines/static-contract.json` cover static contract and dataset integrity. They are not routing-accuracy claims.
- `simulation/cases.json` and `baselines/adapter-simulation.json` execute input, approval, capability, and fallback state transitions without a model.
- `live/cases.json` contains 24 hidden-label public-safe cases. `scripts/run-live-assistant-evals.mjs` defaults to three attempts per case, loads Claude with `--plugin-dir`, loads Codex through sandbox-local `AGENTS.md`, and derives writes/tool use/invented surfaces instead of trusting assistant self-report.
- `surface-ab.json` is generated by `node scripts/evaluate-workflow-surfaces.mjs --write`. It compares static direct-command and compatibility-contract costs; it is not a live latency or model-quality result.
- Live assistant evidence belongs under `evidence/` and must name the assistant, exact version, execution mode, commit, tag, source state, runner/dataset digests, adapter-load proof, result, and skipped capabilities. L4 eligibility requires a clean exact tag, at least 20 cases and 3 attempts per case, and actual adapter loading.

Run `npm ci --ignore-scripts`, then `npm run eval:contracts`, `npm run eval:dataset-integrity`,
`npm run eval:simulation`, and `npm run eval:surfaces` for deterministic gates.
Use `npm run eval:live -- --assistant <claude-code|codex>` only for public-safe
versioned evidence. Use `--case <id>` only for diagnostic smoke; it does not
satisfy the full 20-50 case evidence requirement. Never add private consumer prompts, credentials, endpoints,
or repository names.
