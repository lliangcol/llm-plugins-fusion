# Skill: /review-only
- 来源：`nova-plugin/commands/review-only.md`

## 适用场景
- 系统性评审代码或实现描述。
- 关注正确性、性能、并发与测试覆盖。

## 输入参数
### Required
- 无

### Optional
- `ARGUMENTS`: 待评审内容。示例: `代码片段或实现描述`

## 行为准则（Do/Don't）
### Do
- 按严重性分组输出。
- 区分事实与假设。

### Don't
- 输出实现代码。
- 重写系统或新增需求。

## 详细执行步骤
1. 阅读输入并识别问题。
2. 按严重性分组输出问题。
3. 给出方向性建议但不写代码。

## 输出规范
- 输出按 Critical / Major / Minor 分组，每条包含问题、原因与方向性建议。

## 典型示例
```text
/review-only
请审查以下代码片段：...
```

```text
/review-only
以下是实现描述，请分级输出问题。
```

```text
/review-only
请给出完整修复代码。
```

## 常见误用与纠正
- 误用：提供实现级修复。纠正：仅给方向性建议。
- 误用：扩展到未提供范围。纠正：仅评审已提供内容。
