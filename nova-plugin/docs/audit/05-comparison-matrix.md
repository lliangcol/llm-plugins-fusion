## 5.6 与 Claude Code 未来演进适配度分析

**方案 A**：适配中性。修复文档和测试不影响 Claude Code 加载逻辑，CI 验证插件格式无变化。

**方案 B**：hooks 实现是最关键的与 Claude Code 演进对齐的工作。hooks.json 格式需要跟进 Claude Code hooks 规范（当前 `{"hooks":{}}` 结构仅表示支持该特性但未配置）。如果 Claude Code 扩展 hooks 事件类型，B 方案的实现可以直接受益。

**方案 C**：marketplace 多插件支持与 Claude Code 的插件生态发展方向完全对齐。如果 Anthropic 推出官方插件市场，方案 C 的基础设施可以快速适配。

**方案 D2**：frontmatter 解析构建脚本是内部工具，与 Claude Code 演进无直接关系，但消除双轨维护后，未来命令定义的更新只需修改 commands/*.md，自然与 Claude Code 加载逻辑保持同步。

**方案 D3（MCP Server）**：与 Claude Code 演进高度对齐。MCP 是 Anthropic 正在推进的标准协议，Claude Code 已支持 MCP Server 接入（mcpServers 配置）。将命令构建能力封装为 MCP Server，可以被 Claude Code 直接调用，是最符合"AI-native 工具"趋势的方向。
