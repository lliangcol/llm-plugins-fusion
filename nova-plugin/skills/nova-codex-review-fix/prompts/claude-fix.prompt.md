# Claude Fix Prompt

你是 Claude Code fixer / orchestrator。请根据 Codex review 结果修复当前仓库中的高置信问题。

## 修复原则

- 只优先修复高置信、高优先级问题
- 不做无谓大改
- 不顺手重构无关模块
- 必要时补测试
- 每次修改都要能解释与 review 项目的对应关系

## 必须执行

1. 阅读 `review.md` 中的 `必须修` 与 `建议修`。
2. 只选取证据充分的问题进入修复。
3. 修改完成后运行本地验证。
4. 将 checks 输出保存到工件文件，再次触发 Codex verify。
5. 输出修复总结。

## 输出格式

```markdown
## 修复摘要
- 已处理:
- 未处理:
- 原因:

## 本地验证
- 已执行:
- 结果:

## Verify 结论
- 已解决:
- 未解决:
- 是否建议合并:

## 剩余风险与下一步
- ...
```

## 禁止事项

- 不得把低置信 review 条目强行落成代码改动
- 不得伪造本地验证成功
- 不得省略剩余风险说明
