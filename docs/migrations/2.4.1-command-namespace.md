# 2.4.1 Command Namespace Migration

Status: active migration exception

Claude Code prefixes plugin commands and skills with the plugin namespace. From
`nova-plugin` 2.4.1 onward, public installation and workflow documentation uses
`/nova-plugin:<command-id>` as the stable invocation form.

## User Migration

Replace legacy bare examples such as `/route`, `/explore`, `/produce-plan`,
`/review`, `/implement-plan`, and `/finalize-work` with:

```text
/nova-plugin:route
/nova-plugin:explore
/nova-plugin:produce-plan
/nova-plugin:review
/nova-plugin:implement-plan
/nova-plugin:finalize-work
```

The same namespace rule applies to every advanced command. For example,
`/review-strict` becomes `/nova-plugin:review-strict`.

## 2.4.x Dual Surface

The patch release does not remove either source tree. Claude Code 2.1.205
reports 42 installed Skills:

- 21 thin command entries such as `/nova-plugin:route`.
- 21 compatibility skill entries such as `/nova-plugin:nova-route`.

Use the thin command entry as the public interface. The `nova-*` entry remains
available during 2.4.x for compatibility and implementation tracing; it is not
the recommended first-use surface.

No command, skill, agent, or capability pack is removed by this migration.
