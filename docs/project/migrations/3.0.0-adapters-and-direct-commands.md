# 3.0.0 Direct Commands and Assistant Adapters

Version 3.0.0 raises the maintained Node.js baseline to 22 and makes
`workflow-specs/workflows.json` the canonical workflow source.

- `/nova-plugin:<id>` command adapters execute the referenced contract directly; they no longer invoke a second `nova-*` Skill tool call.
- `/nova-plugin:nova-<id>` remains as a deprecated compatibility surface for one major-version migration window.
- Runtime permissions, route ownership, workflow catalogs, and assistant adapters are generated from the canonical spec.
- Claude, Codex, and generic Agent Skills compatibility claims are separated and evidence-levelled.
- Shared skill rules are referenced once; legacy `$NAME` pseudo-substitutions are represented as logical `<NAME>` values.

Consumers should move invocations to `/nova-plugin:<id>` now. Maintainers must
edit the canonical spec and workflow-specific contract, run both generators,
and record assistant-specific live evidence before raising a compatibility level.
