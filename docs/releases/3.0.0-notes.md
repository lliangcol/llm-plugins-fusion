# 3.0.0 Release Notes

Status: signed immutable tag created; GitHub Release not published

The final publication job rejected a glob passed to the attestation action's
exact `sbom-path` input. The tag remains unchanged under the immutable policy.
Version 3.0.1 supersedes this unpublished release candidate with an exact-path
resolver; no 3.0.0 GitHub Release assets were published.

This release establishes immutable signed SemVer tags, Node.js 22+, direct
command execution, canonical workflow generation, structured validation
evidence, deterministic archives, CycloneDX SBOM, provenance, artifact
attestation, assistant adapters, and public conformance datasets.

## Historical transparency

The public `v2.4.1` ref changed during release-blocker remediation, so that tag
did not always identify one immutable source state. It will not be moved again.
Version 3.0.0 is the first tag protected by the new signed, annotated,
immutable-tag policy. Version 3.0.1 is the first intended complete GitHub
Release under that policy. Existing releases are not retroactively described
as satisfying it.

## Compatibility

- Node.js 22 or newer is required for maintained repository and active guard behavior.
- Direct `/nova-plugin:<id>` names are canonical.
- `nova-*` compatibility names are deprecated but retained for this migration window.
- Claude Code L4 remains contingent on the exact-tag isolated install and live route gates.
- Codex L4 is claimed only when a current exact-version live evidence record exists.
